Hi! I'm Matteo.

I'm a Web & Software Developer currently working for Deltatre, from their Turin offices. Nowadays I work both on the back-end and front-end services, thus I could consider myself a full-stack dev... The languages I use are
.Net/C#, Typescript & React,
with C# being the language I'm mastering as my "main". From a Dbs perspective, I use daily SQL lang. on SQL Server specs and MongoDb on the No-SQL side. To add to the mix of what I always use, there's Git (of course), Azure DevOps & Github for source control management.

Would I like to work with other langs? Sure I would! I'd be happy to study and work with Rust & Go. On the other side, I refuse to work with PHP and Java.
Why? I'm not a jack-of-all-trades fan. I love to learn and master, better than knowing something of everything. Rust or Go would be domains that help me learn and challenge myself on new concepts, different perspectives, low-level topics I didn't study previously. I love to learn, as already said. PHP or Java, on the other side... are C#-like. I like C# more and I prefer to master it, instead of using them all without choosing my bestie. If someone needs PHP, I think it's best to actually hire a PHP dev than making me mock one.

I love programming and I want to grow and learn as much as I can as a developer. I'm trying to learn more about Design Patterns and Software Architecture, to improve my capabilities of creating good, clean, better code. For the same reason, I want to learn and improve over TDD, as I use it but I'd like to become better at it.

Other than these, I unfortunately daily use CSS & Sass - but please ignore this information, the tech thing I utterly hate the most is Css and everything concerning it. If you want me to mostly work on Css-things, well, you're messing with the wrong guy mate. üò§ I used some Docker e Kubernetes but I'm still a near newbie (they're a quite complex new world to approach).
